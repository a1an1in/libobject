# libobject 软件设计说明书

## 1. 引言

### 1.1 编写目的

本文档旨在描述 libobject 面向对象 C 语言基础库的软件架构设计、模块划分、核心机制和接口定义，为软件著作权申请提供技术文档支撑。

### 1.2 适用范围

本设计说明书适用于 libobject v1.0 版本，涵盖核心对象系统、数据结构、网络通信、数据库 ORM、并发编程等模块。

### 1.3 软件概述

libobject 是一个基于 C 语言实现的面向对象编程基础库，提供了一套完整的面向对象编程框架，包括对象系统、数据结构容器、网络通信库、数据库 ORM 框架、并发编程框架等模块。该库通过宏定义和注册机制，在 C 语言中实现了类、对象、继承、多态等面向对象特性。

### 1.4 术语定义

| 术语 | 定义 |
|------|------|
| DEFINE_CLASS | 定义类的宏，自动生成类信息数组并注册 |
| REGISTER_CLASS | 注册类的宏，通过 `__attribute__((constructor))` 在程序启动时自动注册 |
| class_info_entry_t | 类信息条目结构体，描述类的属性、方法等元信息 |
| allocator_t | 内存分配器接口 |
| Orm_Conn | 数据库连接接口，提供统一的数据库操作抽象 |
| cjson_t | JSON 数据结构体，用于 JSON 解析和序列化 |

## 2. 总体设计

### 2.1 设计目标

1. 在 C 语言中实现面向对象编程范式，支持封装、继承、多态

6. 提供脚本引擎集成（FShell 命令行、Python 集成）

## 3. 核心对象系统设计

### 3.1 设计原理

libobject 的核心创新在于通过 C 语言的宏和结构体，实现了一套完整的面向对象编程框架。其核心思想包括：

1. **类信息注册机制**：每个类通过 `DEFINE_CLASS` 宏定义，自动生成类信息数组，并通过 `REGISTER_CLASS` 宏在程序启动时注册到全局类守护者（class_deamon）中。

2. **对象模型**：所有对象继承自 `Obj` 基类，基类提供 `set`、`get`、`to_json`、`reset`、`assign` 等虚函数接口。

3. **属性描述**：类的每个属性通过 `class_info_entry_t` 结构体描述，包含类型、名称、偏移量等信息，支持运行时反射。

4. **虚函数机制**：通过 `Init_Vfunc_Entry` 宏注册虚函数，支持子类重写父类方法，实现多态。

5. **内存管理**：通过 allocator 接口抽象内存分配策略，支持堆分配、池分配等。

### 3.2 类定义宏

```c
// DEFINE_CLASS 宏定义（class_info.h 第110行）
#define DEFINE_CLASS(cls, ...)                                                       \
    typedef cls __current_class_t;                                                   \
    static const char * const current_class_name = #cls;                             \
    static class_info_entry_t cls##_class_info[] = {                                 \
            __VA_ARGS__,                                                             \
            Class_End___Entry(cls)                                                   \
    };                                                                               \
    REGISTER_CLASS(cls, cls##_class_info)
```

### 3.3 类注册机制

```c
// REGISTER_CLASS 宏定义（class_deamon.h 第10行）
#define REGISTER_CLASS(class_name, class_info) \
__attribute__((constructor)) static void register##class_name()\
{\
    __register_ctor_func3(REGISTRY_CTOR_PRIORITY_REGISTER_CLASS,\
            (int (*)(void *, void * , void *))class_deamon_register_class, NULL, #class_name, class_info);\
}
```

### 3.4 对象创建流程

1. `object_new(allocator, type_name, config_str)` 创建对象
2. 从 class_deamon 中查找类信息
3. 分配内存（通过 allocator）
4. 解析 JSON 配置字符串，通过 `object_set` 设置属性
5. 调用 `object_init` 初始化对象
6. 返回对象指针

### 3.5 继承与多态

继承通过类信息链实现：每个类的 class_info 中包含父类的类信息指针。虚函数查找时，从当前类开始，沿着继承链向上查找，找到最后一个重写的实现。

## 4. 数据结构模块设计

### 4.1 Vector（动态数组）

- 支持任意元素类型（通过 value_type 字段指定）
- 支持自动扩容
- 支持 JSON 序列化/反序列化
- 支持自定义元素的 free、to_json、new 回调函数

### 4.2 Map（关联容器）

- 基于红黑树实现的有序映射
- 支持任意键值类型
- 支持自定义比较函数
- 支持 JSON 序列化

### 4.3 String（字符串）

- 可变长度字符串
- 支持格式化操作
- 支持 JSON 序列化

### 4.4 其他容器

- List：双向链表
- Queue：队列
- Stack：栈
- Trie：前缀树
- Linked_List：链表
- Array_Stack：数组栈

## 5. 数据库 ORM 模块设计

### 5.1 架构设计

```
┌─────────────────────────────────────────────┐
│              Application Code                │
├─────────────────────────────────────────────┤
│   Orm (连接池管理)                            │
│   ├── get_conn() / add_conn()                │
├─────────────────────────────────────────────┤
│   Orm_Conn (数据库连接抽象)                    │
│   ├── query() - 执行查询，返回 JSON            │
│   ├── update() - 执行更新                     │
│   ├── exec_sql() - 执行任意 SQL               │
│   ├── insert_model() - 插入模型对象            │
│   ├── query_model() - 查询模型对象             │
│   ├── query_table() - 查询表对象               │
│   └── del() - 删除操作                        │
├─────────────────────────────────────────────┤
│   Model / Table (对象关系映射)                 │
│   ├── Model: 数据模型基类                      │
│   └── Table: 表模型基类                       │
├─────────────────────────────────────────────┤
│   Sql (SQL 抽象层)                            │
│   └── Mysql (MySQL 实现)                     │
└─────────────────────────────────────────────┘
```

### 5.2 核心接口

**Orm_Conn 接口**（Orm_Conn.h 第12行）：

```c
struct Orm_Conn_s {
    // ... 基类成员
    int (*update)(Orm_Conn *connection, char *sql_statement, ...);
    int (*exec_sql)(Orm_Conn *connection, char *sql_statement, ...);
    int (*query)(Orm_Conn *conn, char *json_buf, int buf_size, char *sql_fmt, ...);
    int (*query_model)(Orm_Conn *connection, char *model_name, char *conditions, void *out);
    int (*query_table)(Orm_Conn *connection, char *table_name, char *conditions, void *out);
    int (*insert_model)(Orm_Conn *connection, void *model);
    int (*del)(Orm_Conn *connection, char *table_name, char *conditions);
};
```

### 5.3 Model 与 Table

- **Model**：数据模型基类，描述单条数据的结构，支持 `to_json()` 序列化
- **Table**：表模型基类，包含多个 Model 的 Vector，支持批量操作
